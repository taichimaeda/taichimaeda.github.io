Quote

Non-deterministic finite automata (NFA) is a tuple , where:

  • is a finite set of states
  • is a finite alphabet
  • is the transition function
    • There may be zero to multiple possibilities for each state-input symbol pair
    • Transition may take place on an empty string
      • is the set
  • is the start state
  • is the set of accept state

DFAs must have exactly one arrow exiting each state for the same label.

In contrast, a NFA may have zero, one or multiple arrows exiting each state for the same label:

  • It non-deterministically guesses which arrow is the “correct” transition
  • Its computation can be seen as parallel transitions where at each symbol there exists a set of possible states for the input strings seen so far:
    • Each execution thread “dies” when there are no arrows exiting from the state for the current symbol
    • The NFA accepts string Iff any of these execution threads is in accept state when terminates

In the third proof:

One (slightly bad) idea is simply to add the start state to the set of accept states. This approach certainly adds ε to the recognized language, but it may also add other, undesired strings.

This is because if there exists an arrow into the original start state, the new NFA can accept strings that do not necessarily pass through the original accept states.

Equivalence of NFA and DFA

Quote

NFA and DFA are equivalent

  • i.e. Language is recognised by an NFA iff it is also recognised by a DFA
  • i.e. The set of languages recognised by an NFA and DFA is identical

Proof Idea

  • For , simulate states of NFA with an equivalent DFA
    • The states of the DFA are therefore the power set of the states of an equivalent NFA
    • Make sure to follow all arrows with the empty string label before processing the next symbol
      • This terminates in finite time because the number of states is finite
    • Because the transition is solely based on the current state, the set of possible states is enough to simulate the execution of an NFA
  • For , use the fact that DFAs are a special class of NFAs

For actual proof, see: Introduction to the Theory of Computation, page 79
