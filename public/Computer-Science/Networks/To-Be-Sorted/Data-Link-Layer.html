Data Link Layer

WiFi

Also known as IEEE 802.11.

Wireless access point uses 2.4GHz or 5GHz of the radio spectrum, which are open for unlicensed use. It merely work as a wireless repeater.

Bands

  • ISM band
    • Reserved for industrial, scientific and medical purposes.
  • U-NII band
    • Known as unlicensed national information infrastructure.

These bands can be used by unlicensed purposes and can be prone to interference by:

  • Cordless phones
  • Baby monitors
  • Bluetooth
  • NFC
  • Microwave ovens
  • Amateur radio, etc.

Ethernet

Also known as IEEE 802.3.

Cable Types

  • UTP (unshielded twisted pair)
  • STP (shielded twisted pair; braid screen around cables)
  • FTP (foil twisted pair; foil shield around cables)
  • SFTP (shielded and foil twisted pair)

Twisting reduces the strength of noise signals, and shielding ethernet cables prevents eavesdropping through EMI.

UTP cables were cheap because they were used in telephone systems with mass production.

Cable Pinouts

https://en.wikipedia.org/wiki/Differential_signalling

In Ethernet, there are two pairs of wires used for differential signalling, which means that the data signal is transmitted as a pair of electrical signals with opposite polarities:

  • TX+ and TX-: used for transmitting data
  • RX+ and RX-: used for receiving data

The receiver then looks at the difference between the two signals to interpret the data. Some benefits of differential signalling include double signalling voltage, reduced noise and low power consumption.

Cable Flavours

https://www.computercablestore.com/straight-through-crossover-and-rollover-wiring
https://en.wikipedia.org/wiki/Rollover_cable

  • Straight-through: for communication between different layers (e.g. switch to router)
  • Crossover: for communication between the same layer (e.g. switch to switch)
  • Rollover: for directly tapping into a networking device (e.g. troubleshoot a router from the console port)

A straight-through cable connects TX+/- to TX+/- and RX+/- to RX+/-. A crossover cable connects TX+ to RX+, TX- to RX- and vice versa. A rollover cable has reversed wiring which connects a pc to a router’s serial communication port (using a different standard).

A PC or router uses the MDI specification, while a switch uses MDI-X specification, which comes with a crossover inside it. This is why we need to use a straight-through for connecting switch to pc, and a crossover cable for switch to switch/pc to pc. Check this link for the motivation behind the crossover.

Some modern devices know how to fake-swap between a straight and a crossover using software based approach (Auto-MDI/MDIX).

Frames

An Ethernet frame:

  • Provides well-defined interface to Network Layer for sending/receiving packets.
  • Identifies transmission errors using a Cyclic Redundancy Check (CRC) checksum.

The CRC checksum is stored in the footer.

Mac Address

Every NIC conforming to IEEE 802 has a MAC address:

  • 48 bits written as 6 colon-separated octets in hexadecimal.
  • OUID (first 3 octets) is manufacturer specific
    • Guarantees uniqueness (c.f. ICANN and ISP for IP addresses).
  • I/G bit (last bit in first octet) decides if unicast or multicast address.
    • Unicast means the address is individual
    • Multicast means it is a group address.
  • U/L bit (second last bit in first octet) decides if globally or locally administered.

MAC address can be assigned to a PC, router or switch.

A host MAC address is a unicast and globally administered address, while a broadcast address (FF:FF:FF:FF:FF:FF) is a multicast and local administered address.

Repeater, Hub, Switch and Router

Repeaters, hubs, switches and routers have different purposes and operate in different OSI layers:

  • Repeater
    • Repeats data in LAN.
    • Boosts the signal and extends the range of network.
    • Receives and forwards data from all ports.
    • Wireless version is known as wireless access point (WAP).
    • Physical layer because it only handles signalling.
  • Hub
    • Connects devices in LAN.
    • Broadcasts data to all connected devices.
    • Physical layer because it only handles signalling.
  • Switch
    • Connects devices in LAN.
    • Sends data to the destination device only.
    • Reduces the network congestion.
    • Data-link layer because it needs to read the MAC address in a frame.
  • Bridge
    • Switch with only two ports.
    • Only sends frames connection if the MAC address of the destination matches.
  • Router
    • Connects multiple LANs.
    • Sends data to the destination network.
    • Network layer because it needs to read the IP address in a packet.

A switch uses the FIB MAC table to remember which port is connected to which device by its MAC address.

A router comes with many other features, such as firewalls, NAT and DHCP server.

Switching Methods

  • Store-and-forward
    • Receives the whole frame and forwards it.
    • Slow for large frames.
    • Switch can check the checksum on behalf.
  • Cut-through
    • Start forwarding when you know the header.
    • Fast even for large frames.
    • Recipient must check the checksum.

Switching Loops

A switching loop occurs when there is more than one layer 2 paths between two endpoints (e.g. multiple connections between two switches or two ports on the same switch connected to each other).

Use the following protocols to ensure there are no loops:

  • Spanning Tree Protocol
  • Rapid Spanning Tree Protocol

Network Topology

  • Switched Ethernet Topology
    • Also known as star topology.
    • Single machine connected to each switch port.
    • Sniffing is usually impossible.
  • Inter-networking Ethernet Topology
    • Repeater and switches.
    • Some machines are directly connected to each other via cables.
  • Wireless Network Topology
    • Wireless access points.
    • Sniffing is easy.

Inter-networking Ethernet Topology

Inter-networking Ethernet Topology can be further categorised:

  • Bus Topology
    • BNC T connector to add a new host.
    • BNC terminator to mark the end of network.
  • Ring Topology
    • Two NICs required per host (expensive).
    • Data flow around the ring.
    • A cut in link kills the entire network.
  • Dual Ring Topology
    • Four NICs required per host (more expensive).
    • Faster data flow with backup link.
  • Token Ring Topology
    • Connects all hosts in a logical ring (not necessarily physical).
    • MSAU controlled the mapping from the logical ring to physical network.
    • Frames are passed around the logical ring.
    • Frame are removed from network when reaching the sender.

Token Ring Topology

Token Ring Topology has listen mode and transmit mode:

  • Listen mode
    • Copies frame into buffer.
    • Re-transmits frame to next host.
    • Host can read the buffer at any time.
  • Transmit mode
    • Transmits frames from memory.
    • Drains frames arriving back.
    • Only one host can be in transmit mode.
      • To reduce frame collisions.
    • May release the token before draining all frames (early release).

When there is no data to send, a special frame, called a token, is passed around the circle indefinitely.

A token consists of (pre-defined) start/end delimiter and access control flags. A frame additionally contains the data to be sent, frame control, source/destination MAC address and frame status.

Schemes of Token Ring Topology

  • Priority scheme
    • Host can claim the token if the priority of host is as high as the priority of data.
  • Reservation scheme
    • Host can reserve the right to transmit high-priority data.
    • Host wanting to transmit data can set reservation priority in the current frame (not token).
    • Host transmitting frames sees the frame with reservation priority.
      • When transmission is complete, sends a token with this priority so that other hosts cannot claim the token unless they have higher-priority data.
  • Acknowledgement scheme
    • Frame status A: destination is currently busy.
    • Frame status C: destination correctly read the frame.

Complexities of Token Ring Topology

  • Assumes no one abuses the priority/reservation scheme.
  • Active monitor station is required to:
    • Re-generate lost token.
    • Drain orphan frames.
  • Any host can become active monitor station.
    • Elect new monitor using contention protocol in case of failure.
  • Ring may have a single point of failure depending on the physical topology.
    • Use wiring concentrators to switch out faulty hosts.

Fibre-Distributed Data Interface

FDDI uses optical fibre cabling, which supports data transmission rates over long distances (about 100km).

  • Class A: attached both rings.
    • Any faulty class A will not affect the remaining class As.
  • Class B: attached to only one ring.
    • Any faulty class B affects one ring at maximum.

Medium Access Control

MAC is necessary to resolve frame collisions in broadcast channels, which comes with higher chance of frame collision.

  • In a wired network, frame collision result in both frames needing to be re-transmitted because the voltage is almost the same within a LAN.
  • In a wireless network, one signal may be stronger/weaker and a frame may be received even with collision.

Collisions are usually detected if broadcasting host does not receive an acknowledgement (ACK) for the frame after a certain amount of time.

Collisions are inevitable as there is no central authority controlling transmission. It is a best-effort service, in the worst case a frame may be indefinitely delayed.

  • Suitable for most LANs
  • Unacceptable for real-time systems.

Strategies for Medium Access Control

  • No control
    • Very inefficient.
    • Good if channel utilisation is low.
  • Round-robin
    • Stations take turns using the channel.
    • Used in token-based MAC systems
  • Reservations
    • Stations need to obtain channel reservation before transmitting.
    • Used in slotted MAC systems.
  • Token-passing
    • Token transferred from one station to another.
    • If a station has the token but no frame to send, pass token on immediately.
    • If a station has the token and a frame to send, it sets a timer and transmits until the timer expires or there is no more data to send.
  • Using switches
    • Each Ethernet cable acts as a channel, which has at most two stations (host and switch).
    • Switch ensures that collisions never occur by buffering frames and re-transmitting when the channel is available.
    • Switches act as repeaters, refreshing the signal to pass it further.
  • Static channel allocation
    • Time Division Multiplexing (TDM)
    • Frequency Division Multiplexing (FDM)
      • Not ideal if many stations exist.
  • Dynamic channel allocation
    • ALOHA protocol
      • Stations may transmit at any time.
      • When collision occurs, wait for a random time interval before retransmitting.
      • Fair channel access.
      • Low channel efficiency.
      • Large vulnerable period (i.e. new frames can easily destroy old frames).
    • Slotted ALOHA protocol
      • Stations may transmit only at the start of a specific time slot.
      • Small vulnerable period (i.e. collision only happens with exact overlap, assuming the time slot is long enough).
      • 100% load gives 36% efficiency.
    • Carrier sensing
      • Stations may transmit only when channel is idle.
      • Collisions can still occur due to transmission delay.

Collision Detection for Medium Access Control

In collision detection, host listens to channel during transmission to check for collisions.

  • Transmission stops when collision detected, then sends a jamming signal to notify other stations of collisions.
  • Stations must transmit long enough to check if the frame collided.
    • Minimum frame length is where is a end-to-end transmission delay.

The minimum frame length for Ethernet is 512 bits. There a few ways to work around this limitation for small frames:

  • Career extension
    • Any frame less than 512 bits is extended by the host.
  • Frame bursting
    • Several short frames can be used to make one long frame.

Channel Back-off for Medium Access Control

  • 1-persistent CSMA
    • Station keeps checking if the channel is free and then transmits immediately.
  • -persistent CSMA
    • Station keeps checking if the channel is free and then transmits at probability .
  • Non-persistent CSMA
    • If the channel is busy, wait for a random amount of time before checking again.
  • Binary exponential back-off
    • Minimum frame length is called a slot.
    • If a collision occurs in transmission, wait 0 or 1 slots before attempting again.
    • After collisions, wait to slots, up to the limit of 1023 slots / 10 collisions.

Ethernet uses 1-persistent CSMA when network load is low, and binary exponential back-off when network load is high.

Address Resolution Protocol

A MAC address is enough for communication within a LAN, but an IP Address is required to communicate outside of the LAN.

  • IP addresses specify addresses of hosts involved in end-to-end communication
  • MAC addresses specify addresses of hosts involved in end-to-end communication within the same network.

Switches are in the data-link layer and cannot read the destination’s IP addresses in network layer, so they send an ARP request to routers to find out the MAC address.

IP Address can be set statically (fixed IP, set manually) or dynamically (assigned to your NIC, e.g by DHCP service).
IP Addresses specify hosts on the internet, it does not have to be translated when passing through a router (but can, e.g NAT).
MAC Addresses specify hosts communicating on the same network/subnetwork. Typically do not change when passing through routers (as packets).
