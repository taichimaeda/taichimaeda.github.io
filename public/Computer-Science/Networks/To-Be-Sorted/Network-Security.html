Network Security

Access Control

Assuming a secure channel, the guard controls:

  • Which principals can access the resource .
  • Where principals are allowed to be located.
  • What requests principals are allowed to make.

Firewalls

Firewalls control access to the network, which acts as a security gateway between the internal and external networks.

  • Application level gateway
    • Runs on the host
    • Protects the host only.
  • Proxy server
    • Runs on the network.
    • Protects the entire LAN.
  • Circuit-level gateway
    • Session-level control
  • Packet filtering
    • Stateless: checks source/destination IP addresses and ports.
    • Stateful: remembers connections and checks if packets are consistent.

Firewalls can be purely software-based, or even hardware-based.

Firewall Modes

  • Normal
    • Client is aware of the proxy
    • Set up needed on client.
  • Transparent
    • Client is unaware of the proxy.
    • No set up needed on client.
    • Local router/proxy server takes care of forwarding data.
  • Reverse
    • Runs on the outgoing traffic.
    • e.g. CDN load balancing on behalf of CDN DNS servers.

Bastion Hosts

Bastion hosts expect to be attacked and performs auditing/logging when necessary.

  • Should run a trusted and secure OS:
  • Runs minimal software and services:
    • Remove non-essential applications.
    • Set file systems read-only.
    • Turn on file quotas and process limits.
    • No regular user accounts.
    • No NFS mounts.
  • Administered via a dedicated terminal.

Application-level Gateways

  • Can block, filter and report based on application-level message content.
  • Can scan for data leaks, worms/viruses, etc.
  • Can even rewrite data.

Stateful Connection Firewalls

  • Can authenticate users.
  • Can drop connections based on destination, incorrect connection packets, time, volume, etc.
  • Useful for logging/auditing/monitoring.

Packet Filtering

Access control list is a collection of packet filtering rules:

  • Rules are checked from top to bottom until a match is found.
  • The last row usually blocks all the remaining traffics.

IDS and IPS

  • Intrusion Detection System (IDS)
    • Software that detects intrusions (e.g. identifies a DDoS attack).
    • Only informs the system.
  • Intrusion Prevention System (IPS)
    • Software that prevents intrusions (e.g. actively blocks SYN flooders).
    • Includes or works with an IDS
  • Next Generation Firewall (NGFW)
    • Stateful firewall with IPS/IDS system
  • Unified Threat Management (UTM)
    • Similar to NGFW but with more capabilities (e.g. anti-spam/antivirus).

Port Forwarding

Port forwarding is a technique used to allow incoming traffic to a public IP address/port to be forwarded to a private IP address/port.

This can be useful for:

  • Hosting a web server without exposing private IP.
  • Hosting a game server running on non-standard ports blocked by firewall.

Port forwarding is a specific application of NAT that only handles incoming traffic.

SSH Tunneling

SSH tunneling allows a user to bypass the firewall:

  1. Establish a SSH connection to an intermediate host behind the firewall.
  2. Connect to the target host from the intermediate host.
    1. This works because the intermediate host is in the same network as the target host.

Kerberos

Kerberos is a user authentication system which holds your passwords.

It is key distribution centre that can provide a ticket, allowing you to communicate with the user/resource. Tickets expire after a predefined amount of time.
