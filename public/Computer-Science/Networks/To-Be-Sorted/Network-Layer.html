Network Layer

Internet

The Internet is a collection of autonomous systems connected together by backbones.

Internet Protocol

IP data are fragmented into datagrams, which are routed through the Internet:

  • Routers pass IP datagrams to the underlying data link layer (e.g. to LANs and WANs).
  • Switches pass Ethernet frames to the underlying physical layer.

Internet Protocol specifies:

  • Datagram format
  • Fragmentation
  • Addressing
  • Packet handling

Fragmentation in IP

Routers must handle cases where size of input datagram exceeds the MTU, which is identified by Path MTU Discovery. Datagrams may need to be fragmented further along the path.

The destination reassembles fragmented datagrams, not the intermediate routers.

IPv4 Address

IP addresses are managed by nonprofit ICANN to avoid conflicts.

ICANN delegates parts of address space to various regional authorities, which give out IP addresses to ISPs and companies.

Special IP addresses:

  • 0.0.0.0/0
    • Represents any IP address.
    • Used in firewalls and servers to listen on any IP address.
    • Used in routing table for the default entry.
  • 127.0.0.0/8
    • Loopback address.
    • Can send packets to this address.
  • 169.254.0.0/16
    • Returned to indicate the absence of DHCP servers.

IP address can be assigned to a PC or router.

Each IP address is associated with an NIC, so it is possible for one host to have multiple IP addresses or share a single IP address with multiple hosts.

IPv4 Headers

  • Version (4 bits)
  • Internet Header Length (4 bits)
    • Size of IP header in 32-bit words.
  • Type of Service (8 bits)
    • Specifies how datagram should be handled by routers e.g. priority level.
  • Total Length (16 bits)
    • Total length of the packet in bytes (including the header).
  • Identification (16 bits)
    • Unique value assigned to each datagram used during fragmentation and reassembly.
  • Flags (3 bits)
  • Fragment Offset (13 bits)
    • Position of a fragment in the original un-fragmented packet.
  • Time to Live (8 bits)
    • Maximum number of hops a packet can take before it is discarded.
  • Protocol (8 bits)
    • Protocol used in the data portion of the packet.
  • Header Checksum (16 bits)
    • Checksum for header.
  • Source IP Address (32 bits)
  • Destination IP Address (32 bits)
  • Options field (variable)
  • Padding field (variable)

IPv6 Addresses

IPv6 keeps the main header as simple as possible and provides further information in optional extension headers.

  • Version (4 bits)
  • Traffic Class (8 bits)
    • Used to prioritise packets.
  • Flow Label (20 bits)
    • Used to identify packets that belong to the same flow.
  • Payload Length (16 bits)
    • Length of the packet’s payload (excluding the header).
  • Next Header (8 bits)
    • Protocol type of the next header after the IPv6 header.
  • Hop Limit (8 bits)
    • Similar to the Time to Live (TTL) field in IPv4.
    • Limits the number of hops a packet can take before being discarded.
  • Source Address (128 bits)
  • Destination Address (128 bits)

The following fields have been removed from IPv4:

  • Fragmentation
    • IPv6 pushes fragmentation onto end-systems for efficiency.
  • Header checksum
    • Redundant as IP header is small.
    • Available in extension headers.
  • Options
    • Fixed-length headers are easier to process.
    • Extension headers allows better modularity.

Routing Hierarchy

All interfaces in same subnet share the same address prefix so that routers outside the network can easily identify the subnet.

  1. External routers only consider network address and forward the packet to one of the routers of the organisation.
  2. Subnet routers apply the subnet mask, and look up if the destination is on their subnet, or if they should forward it to another subnet router
  3. After host was found, router knows which interface to use to forward packet

Notations for addressing subnets:

  • Class notation
    • Class A: 8 bits
    • Class B: 16 bits
    • Class C: 24 bits
  • Prefix-length notation: address/prefix-length.
    • Known as Classless Inter-Domain Routing (CIDR).
  • Mask notation: address/mask.

The smallest IP address in the subnet is used for identifying the subnet itself (e.g. for routing). The largest IP address in the subnet is the broadcast address for the subnet.

Routing

  • Routers must know the topology of the network to choose appropriate paths.
  • Routers must balance loads among routes.
  • Routers also has to deal with network heterogeneity.

Routers find the network address using subnet mask, and forward the packet based on the information found in the forwarding table.

The longest prefix match is used when the forwarding table contains multiple subnets that contain the destination IP address.

Routing Algorithms

Routers collaborate to build a sink tree of each node, which only contains the optimal routes towards that specific host.

Shortest Path Routing

Forward packets along the shortest path using Dijkstra’s algorithm.

Flood Routing

Forward packets across every outgoing link, except the original source.

Flood routing always chooses the shortest because all paths are explored in parallel, but the overhead is significant.

Some ways to minimise the overhead include:

  • Use a hop counter
    • Discard packet after forwarded times.
    • Difficult to find the appropriate hop count.
  • Forward packets only once per router
    • Avoid building directed cycles.
    • Requires router to record packets it forwarded.
  • Flood selectivity
    • Forward packets only in some direction.

Distance Vector Routing

Distance vector routing is implemented by RIP protocol.

Forward packets along the shortest path using a variant of Bellman-Ford algorithm.

Distance vector routing is a dynamic protocol that takes network load into account.

Link state routing is implemented by OSPF protocol.

  2. Advertise to all other nodes.

  3. To forward packet
    1. Run Dijkstra’s algorithm locally.
    2. Select neighbour on the shortest path.

A well-known issue is that routers may redirect traffic towards the best routes so much these routes become overloaded.

Broadcast Routing

Broadcast routing aims to send message to almost every host on the network

Multicast Routing

Multicast routing aims to send message to only a group of hosts on the network.

  1. Select core for each group.
  2. Source host sends message to the core whose group contains multicast targets.
  3. Core sends message to its group.
    1. Use broadcast routing with pruning.

Hierarchical Routing

Issues with flat network model:

  • Scalability
    • Hundreds of millions of hosts in today’s Internet
    • Transmitting routing information too expensive
    • Forwarding also too expensive
  • Administrative autonomy
    • Organisations may want to use different routing protocols within their network.
    • Organisations do not want to expose internal network structure

Hierarchical routing aims to find suboptimal routes by introducing regions, and separate algorithms for intra-region and inter-region routing.

Gateway routers interconnect autonomous systems.

  • Intra-AS routing protocol
    • Should provide optimal routing.
  • Inter-AS routing protocol
    • Only provides suboptimal routing.
    • e.g. Hot-potato routing, BGP

In hierarchical routing, both inter-AS and intra-AS routing information used to populate forwarding tables.

Open Shortest Path First

OSPF allows an AS to be divided into areas.

  • Use the same algorithm for the areas and the backbone.
  • One or more area border routers responsible for routing packets outside its area.
  • Backbone area routers redirect traffic between other areas in AS.

Border Gateway Protocol

BGP is path-vector protocol:

  • Based on distance vector routing.
    • But path vectors are advertised instead of distance vectors.
  • Does not suffer from count-to-infinity problem.
    • Router makes decision on the entire path.
  • Routers may aggregate prefixes (a.k.a. super-netting).

In BGP, neighbouring routers uses TCP connection to simplify reliability.

  • Provides reachability information from neighbour AS.
  • Transmits reachability information to all internal routers within AS.

Some BGP attributes:

  • ASN
    • Autonomous System Number.
    • Unique identifier for each AS.
  • AS-PATH
    • Sequence of ASNs through which advertisement was sent.
    • Advertised per destination AS.
  • NEXT-HOP
    • Specifies IP address for forwarding packets towards advertised destination.
      • Useful for when AS is reachable through multiple IP addresses.
    • Advertised per destination AS.

BGP import policy decides whether to accept or reject route advertisement
e.g. router may not want to send traffic through one AS listed in AS-PATH.

Dynamic Host Configuration Protocol

Internet users receive an IP address from ISP via DHCP:

  • Each newly-booted machine broadcasts a DHCP DISCOVER packet.
  • DHCP server replies with the assigned IP address.
  • DHCP server can either maintain static mappings or assign different addresses each time a host connects.

Network Address Translation

NAT translates a single public IP address into many private IP addresses. It holds the mapping from the public port to the private IP address and its port.

  • Within a LAN, every computer gets a unique private IP address for routing internal traffic.
  • Address translation takes place when a packet exits the LAN.

The following address ranges are declared as private for use in local networks:

  • 10.0.0.0 – 10.255.255.255/8 (16,777,216 addresses)
  • 172.16.0.0 – 172.31.255.255/12 (1,048,576 addresses)
  • 192.168.0.0 – 192.168.255.255/16 (65,536 addresses)

There are some criticisms of NAT:

  • NAT violates architectural IP model, which states that every IP address uniquely identifies single machine worldwide.
  • NAT changes Internet from connectionless to connection-oriented network.
    • NAT gateway must maintain mapping for each connection.
  • NAT violates fundamental rule of protocol layering.
    • NAT in the network layer is dependent on TCP in the transport layer.
  • Many P2P protocols require full connectivity among hosts.
    • Users behind NAT cannot be contacted directly by other peers.
    • Techniques exist to circumvent these issues (e.g. port forwarding).

IPv6 is the best solution to deal with the shortage of IP addresses.

Internet Control Message Protocol

ICMP is used to inform hosts and routers about errors and send queries about status information (e.g. ping).

  • Encapsulates control messages in IP packets .
  • Message type is specified by code (e.g. destination unreachable = 3).
