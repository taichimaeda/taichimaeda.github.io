OSI Model

5-Layer OSI Model

  • Application Layer (Layer 5)
    • Defines applications functionality and message formats.
    • e.g. DNS, SMTP, FTP, Telnet, NNTP, HTTP/HTTPS, P2P (BitTorrent)
  • Transport Layer (Layer 4)
    • Offers connection-oriented and connectionless services.
    • Responsible for data order, reliability and flow control.
    • Provides network interface through sockets.
    • Runs on end systems only (not on routers/switches).
    • Data is called segments.
    • e.g. TCP, UDP, QUIC
  • Network Layer (Layer 3)
    • Describes routing procedures.
    • Responsible for multicasting and congestion control.
    • Runs on intermediate systems.
    • Data is called packets.
    • e.g. IP, ICMP, DHCP, RIP, OSPF, BGP
  • Data Link Layer (Layer 2)
    • Describes frame formats.
    • Responsible for transmission error detection and medium access control.
    • Data is called frames.
    • e.g. Ethernet
  • Physical Layer (Layer 1)
    • Describes transmission of raw bits.
    • e.g. DSL, ADSL

Types of Network

  • Personal Area Network (PAN)
    • e.g. Bluetooth
  • Local Area Network (LAN)
    • e.g. Home WiFI
  • Metropolitan Area Network (MAN)
    • e.g. Free City WiFi
  • Wide Area Network (WAN)
    • e.g. Internet

Packet Switching and Circuit Switching

The Internet is a packet-switched network, where information is transmitted in packets. In contrast, the telephone network is a circuit-switched network, which involves the following procedures:

  • Communication requires a connection setup phase.
  • Communication links are dedicated to their connection for the entire duration of conversation.
  • When the conversation ends, the connection is destroyed.

Circuit switching requires an expensive setup phase, but once the connection is established, little or no processing is required. Packet switching does not incur any setup cost, but it always incurs a significant processing and space overhead.

Circuit switching allows only a limited sharing of communication resources, while packet switching achieves a much better utilisation of network resources.

Connection-oriented and Connection-less

  • Connection-oriented
    • Establish a connection, exchange data and release the connection.
    • Circuit switching is a form of connection-oriented protocol.
  • Connection-less
    • No connection is involved.
    • Data is sent in a packet that contains information required for delivery.
    • Packet switching is a form of connection-less protocol

Communication Protocols

A communication protocol is an agreement between the communicating parties on how communication is to proceed.

Phases of a common communication protocol:

  • Handshake: establishes the context
  • Conversation: free-form exchange
  • Closing: terminates the conversation

Examples of issues on communication protocol design include:

  • Addressing: how to denote the intended recipient
  • Error Control: how to detect and fix transmission errors
  • Flow Control: how to prevent a fast sender from swamping a slow receiver
  • Multiplexing/Demultiplexing: how to support multiple communications in parallel
  • Routing: how to route packets to the final destination

Protocol Layering

A service is set of primitives that a layer provides to the layer above it. The same service can be realised by different protocols.

  • Layer puts its entire packet as data into a layer packet. The layer packet may contain a header and/or a footer.
  • Layer data may be split across several layer packets (fragmentation).
