Editing Past Commits

Imagine we have a commit graph like this:

A---B---C (feature, origin/feature)

and we want to edit the commit B.

Rebasing a Temporary Branch

amending a single file in a past commit in git - Stack Overflow

  1. Create a temporary branch e.g. temp from the commit B.
A---B (temp)
	 \
	  C (feature, origin/feature)
  1. Edit the files.
  2. Add the files to the index and amend the commit B.
A---B' (temp)
 \
  B---C (feature, origin/feature)
  1. Rebase the feature branch onto the temp branch.
A---B' (temp)
 \   \
  B   C (feature, origin/feature)
  1. Delete the temp branch.

Using git commit --fixup

git commit —fixup が便利Git - Qiita

fixup is a command you can use in interactive rebase.
It is like squash in that it melds into the previous commit, but keeps only the previous commit’s commit message.

Given a commit history like this, assume we want to edit commit Add feature 1:

9c57cb3e7a Add feature 2
ab6a6bef9d Add feature 1
cgbaeef9d3 Initial commit
  1. Run git commit --fixup ab6a6bef9d to create a new !fixup commit:
26a2dc16a8 !fixup Add feature 1
9c57cb3e7a Add feature 2
ab6a6bef9d Add feature 1
cgbaeef9d3 Initial commit
  1. Run git rebase -i --autosquash <commit>~1 to squash Add feature 1 and the corresponding !fixup commits:
1131338365 Add feature 2
9c5a2d3e7a Add feature 1
cgbaeef9d3 Initial commit

Tip

--autosquash squashes commits whose commit message starts with !fixup, and melds them into the commit with the same commit message.

This is also useful during PRs - reviewers can check the !fixup commits, and squash them when merging:
git commit —fixupを使いましょう - Don’t Repeat Yourself (blog-dry.com)
